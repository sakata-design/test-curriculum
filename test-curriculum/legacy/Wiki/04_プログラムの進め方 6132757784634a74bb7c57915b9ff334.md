# 04_プログラムの進め方

# テスト設計の対象サービス

◼️対象サービス

- [DockLoad](https://docker-project-a4dda.web.app/)

◼️サービス概要

- 構築済みのDockerファイルがダウンロードできるサイト

◼️設計書フォルダ

- [DockLoad設計書](https://drive.google.com/drive/folders/1i-1hoYM7VGtccPuDD8cIBPpkkiRe34N2?usp=drive_link)

# 課題について

上記のDockLoad設計書をもとに、テスト設計を行なっていただきます。

◼️成果物：以下5つの機能の単体試験紙を作成してください。

- TOP画面
- 導入手順画面
- 構築手順画面
- プライバシーポリシー画面
- お問い合わせ画面

なお当サービスは本番リリースしているものとなりますので、実際にテストデータの作成ができません。そのため、DockLoad設計書に本番に登録されているデータが記載している「テストデータ資料」のスプレッドシートに記載しておりますので、テスト設計する際には、こちらを参考にしてください。
