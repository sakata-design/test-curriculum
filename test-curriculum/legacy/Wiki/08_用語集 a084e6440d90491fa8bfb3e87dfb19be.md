# 08_用語集

# テスト用語

- **ハードウェア**
パソコン本体やディスプレイなど物理的な機器を指します。
- **ソフトウェア**
コンピュータを動かすプログラムの総称で、パソコンや機器に読み込ませることで作業を実行させることができるデータを指します。
- **ドキュメント**
一般には文書や記録、書類のことです。
仕様解説書や説明書、設計書、フローチャートなどのことを指します。
- **アプリケーション**
ソフトウェアの一つで、特定の作業の目的に応じて設計されたもののことを言います。
- **モジュール**
パソコンやタブレットなどの[ハードウェア](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)は、CPUやメモリ、入力装置などの個別に機能を持つ部品の組み合わせで成り立っており、この一つひとつがモジュールに該当します。
また、[ソフトウェア](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)にはさまざまな機能が組み込まれていますが、この機能の一つひとつをモジュールと呼びます。
- **インターフェイス**
パソコンとマウス、WebサイトとGoogleマップといった、特定の異なる2つのシステムなどを結ぶ機器、システムのことを指します。
- **外部インターフェイス**
あるシステムやプログラムが他のシステムやプログラムとデータをやり取りするための機能や手段のことを指します。
- **ログ**
コンピュータの利用状況やデータ通信など履歴や情報の記録を取る事、またその記録を指します。
操作やデータの送受信が行われた日時と行われた操作の内容や送受信されたデータの中身などが記録されます。
- **デバッグ**
コンピュータのプログラムなどのバグや欠陥を見つけ、修正していく作業のことを指します。
- **パッチ**
プログラムの修正や機能変更を行うデータのことを指します。
また、パッチを用いてデータを修復することを「パッチを当てる」などと表します。
- **データベース（DB)**
データを1つの場所に集約し、保管・管理だけでなく、参照したいデータを簡単に抽出できるようにする「箱」のようなシステムです。
- **SQL**
データベース言語の一つで、データベースの定義や操作を行うことができます。
SQLはISO（国際標準化機構）で規格が標準化されており、一度学習すればほかのデータベースでもほぼ同じように操作可能です。
- **要件定義書**
非常に大まかに言えばITシステムやWebサイトによって実現したいことを定めた[ドキュメント](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)です。
結果として、システム開発プロジェクトにおけるゴールを定めるドキュメントにもなっています。
- **画面定義書**
システムで表示される画面のレイアウトを定義している成果物のことです。
画面定義書の作成は、クライアントと開発者の認識のズレを減らすという目的があり非常に重要な成果物の一つです。
また開発者がどのような画面を作成すれば良いかを理解しやすくする役割もあります。
- **テストケース**
ソフトウェアやアプリケーションを詳細にテストするための手順や条件のセットです。
特定の機能、または全体のシステムが正しく機能し、要件を満たしているかどうかを確認するために使用されます。
- **GUI**
Graphical User Interfaceの略称。
WindowsやMacのアプリのようにマウスでアイコンなどをクリックして操作できるシステムのことです。
- **CUI**
Character User Interfaceの略称。
画像やアイコンを使わず、キーボードだけで操作する表示システムのことです。
- **プロトコル**
コンピュータでデータをやりとりするために定められた手順や規約、通信における送受信の手順などを定めた規格を意味します。
- **API**
Application Programing Interfaceの略称。
接続先のOSを呼び出すことや互いのソフトウェアやアプリケーション機能の一部を共有することです。
- **アジャイル開発**
実装とテストを短期間・小さな単位で繰り返し、スピーディーに開発する方法です。
- **ウォーターフォール開発**
滝の水が上から下へ流れ落ちるように、一つ一つの工程を完了させながら進める開発方法です。
品質担保やスケジュール管理がしやすい一方で、急な仕様・要望変更に対応しにくいという特徴もあります。
- **マイグレーション**
ソフトウェアやシステム、データなどを別の環境に移転したり、新しい環境に切り替えたりすることです。
- **デグレーション**
プログラムの不具合を修正したときに、ほかに新たな不具合が出てくることを言います。
 品質や機能が修正前より悪い状態になることです。
- **ステージング環境**
本番環境と同じようなテスト環境のことを指し、いわゆる最終テストを行う環境のことです。
実際にユーザーが使用する環境と類似した環境でテストを行うため、これまでの段階で見つけられなかったエラーや問題点に気づきやすいのが特徴です。
- **単体テスト（ユニットテスト）**
プログラムを組んだ[モジュール](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)やコンポーネントといった機能単位の動作などを単体で確認するテストです。
- **結合テスト**
単体テストでテストされたモジュールを連携した状態で正しく動作するかどうかを検証するテストです。
モジュール単位では期待通りに動作しても、それぞれを連携させるとうまく動作しないことが発生します。
- **ホワイトボックステスト**
ソースコードが明らかな状態で行うテストです。
対象となるプログラムの内部構造に着目するため、テスト実施者にはプログラミングに関する知識が必要不可欠です。
- **ブラックボックステスト**
ソースコードが不明な状態で行うテストです。
テスト対象に期待する動作（仕様書で求められている動作）を検証するもので、モジュール内でデータがどのように処理されるには着目せず、入力と出力の整合性にのみ着目します。
ブラックボックステストにはコーディングのスキルは不要で、ソフトウェアテストのスキルがあれば実施できます。
- **総合テスト（システムテスト）**
個々のモジュールを対象とした単体テスト、複数のモジュールを組み合わせた結合テストがすべて終わったあとに仕上げとして行うテストです。
製品として完成したものを本番とほとんど同じ環境でテストします。
- **回帰テスト（リグレッションテスト）**
機能追加やバグ修正などでコードの一部が改修されたあと、それまでの動作に意図しない変更や問題が生じていないか確かめるテストです。
- **受け入れテスト**
完成したソフトウェアが発注者の期待にそっているかを検証するテストです。
アプリケーションをブラックボックス的に動作検証するテストですが、実施の主体がアプリケーションの発注者側になります。
- **負荷テスト**
アプリケーションが高負荷となった場合の挙動を検証するテストです。
一般的にはサーバー上で動作するアプリケーションに対して多数のユーザーが同時にアクセスした場合を想定して行います。
- **モンキーテスト**
対象箇所や操作手順などを事前に定めず、実施者がその場の思いつきで操作してみるテストです。
- **内部結合テスト（ITa）**
プログラム内部の部品間の結合を検証するテストです。プログラムの部品とは、関数やモジュールのことを指します。
- **外部結合テスト（ITb）**
プログラム内部ではなく、外部とのインターフェース上で発生する問題を特定するためのテストです。
外部とは、データ入出力やネットワークといった部分を指します。
実際の業務フローを再現し、外部とのインターフェースにおいて問題がないかを確認します。

# ツール

## 業務ツール

- **Excel　スプレッドシート**
テスト仕様書やテストケース作成、エビデンスを管理するために使うツールです。
ショーカットキーが使えると作業効率UP！
- **VSCode**
ソースコードの編集を主な目的としたテキストエディタです。
- **TeraTerm**
ネットワーク経由でサーバーの遠隔操作が可能になります。
サーバーの接続にはTelnet・SSHの通信[プロトコル](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)が利用可能です。またシリアル接続にも対応しています。
Windowsにのみ対応しているソフトウェアです。Macをはじめ他のOSでは利用することはできません。
- **A5:SQL Mk-2**
A5:SQL Mk-2 はデータベース開発において生産性を向上させるべく開発された[SQL](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)開発ツールです。
SQLの実行・テーブルの編集などを行うことが出来ます。
- **WinMerge**
オープンソースのWindows 用比較・マージツールです。
フォルダとファイル両方が比較でき、理解しやすく扱いやすいテキスト形式で差異をビジュアルに表現します。
- **Git**
ファイルの変更状況を追跡・管理できるシステムです。
システムを利用するためには、コマンドを使って命令を出す必要があります。
- **SourceTree**
Gitをコマンドではなく[GUI](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)で扱うことができます。
- **コマンドプロンプト**
Windowsに搭載されているシステムツールで「キーボードだけで操作する画面インタフェース」（[CUI](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)）のことを言います。
「コマンド」と呼ばれる命令文を入力して操作を実行します。
- **ターミナル**
 Macに搭載されているシステムツールで「キーボードだけで操作する画面インタフェース」（CUI）のことを言います。
「コマンド」と呼ばれる命令文を入力して操作を実行します。
- **Charles**
HTTPプロキシ、HTTPモニタを行うためのツールです。
簡単にいうと通信時のリクエスト、レスポンスの中身を確認したり、値を書き換えたりすることができます。
- **Postman**
[API](08_%E7%94%A8%E8%AA%9E%E9%9B%86%20a084e6440d90491fa8bfb3e87dfb19be.md)の開発、テスト、ドキュメント作成などを支援するためのツールです。
- **Visual Studio**
Visual Studio は、マイクロソフトが開発した IDE （統合開発環境）です。 
C# 、 VB 、 NET 、 C++ など多くのプログラミング言語に対応しており、Windows アプリケーション、Web アプリケーションなどさまざまな種類のアプリケーション開発をサポートしています。
- **Xcode**
Appleが提供している統合開発環境（IDE）です。
テキストエディタ、コンパイラ、リンカ、デバッガなど開発に必要なソフトウェアを１つにまとめた開発環境のことです。
iPhoneやiPad、MacなどのApple製端末で使用するアプリを作成する際に使用します。

## プロジェクト管理ツール

タスク管理、バグトラッキング、リソース管理、ドキュメンテーションなどに使われるツールです。

- **backlog**
ガントチャートでタスクの進捗やプロジェクトの納期を把握できるので、プロジェクト管理がしやすいです。
GitやSubversionのリポジトリ機能も搭載しており、ソースコードやコミット履歴を管理できます。
- **Redmine**
チケットという概念でタスクを管理しています。
チケットの発行によってタスクをプロジェクトメンバーに割り振れるようになっており、タスクの詳しい内容の付加も可能です。
- **Jira Software**
チーム全体のタスクやスケジュールをユーザーが使いやすいように管理できるカスタマイズ性の高いツールです。
そのため、複数のチームが並行してプロジェクトを進めるアジャイル方式で開発を行う場合にも管理しやすいです。
- **TestRail**
テストケースに必要なデータを添付するなどして、テストに関わる情報を一元的に管理することが可能です。 
優先度などによって目的のテストケースを簡単に検索できるため、スピーディーにテストを実行できます。

## コミュニケーションツール

ビジネスチャットや会議機能を持つコミュニケーションツールです。

- **Slack**
外部連携やテキストベースのコミュニケーションの場合に真価を発揮するなどのメリットがあります。
- **Teams**
Office アプリの連携や豊富な会議機能を持つサービスです。
- **Chatwork**
タスク管理機能を持っており、チャットをしながら発生したタスクを即座に作成・管理できます。
- **Google Workspace**
ローカルにデータを保存しておけば、オフラインでも編集でき、次回接続時に自動アップロードできるという便利な機能もあります。
